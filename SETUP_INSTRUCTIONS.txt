API ENDPOINTS:
================================================================================

Authentication:
  POST /api/auth/start-auth           Generate OAuth URL
  POST /api/auth/oauth-callback       Exchange code for tokens

Gmail:
  POST /api/gmail/send-email          Send email
  GET /api/gmail/list-emails          List emails

  GET /api/sheets/read-sheet          Read spreadsheet

GOOGLE OAUTH SETUP:
================================================================================

1. Go to Google Cloud Console: https://console.cloud.google.com/
2. Create a new project
3. Enable these APIs:
   - Gmail API
   - Google Sheets API
   - Google Drive API
4. Create OAuth 2.0 credentials (Web application)
5. Add redirect URI: http://localhost:3000/callback
6. Download credentials JSON
7. Get Client ID and Client Secret
8. Use in /api/auth/start-auth endpoint

DEPLOYMENT OPTIONS:
================================================================================

1. VERCEL (Recommended for serverless)
   - Push to GitHub
   - Import in Vercel
   - Set environment variables
   - Deploy
   - Time: 5 minutes

2. AWS LAMBDA
   - Install Serverless Framework
   - Configure serverless.yml
   - Deploy with serverless deploy
   - Time: 15 minutes

   - Build: docker build -t google-services .
   - Run: docker run -p 3000:3000 google-services

   - SSH into Linux server
   - Clone repository
   - npm install
   - Set up PM2
   - Configure Nginx
   - Time: 20 minutes

ENVIRONMENT VARIABLES:

Development:
  PORT=3000

  CORS_ORIGIN=https://yourdomain.com
  TOKEN_STORAGE=mongodb
  MONGODB_URI=mongodb+srv://user:password@cluster.mongodb.net/db

DATABASE SETUP:
================================================================================

Development:
  - In-memory storage (default)
  - No database required
  - Tokens lost on server restart

Production:
  - MongoDB: Recommended for cloud
    Connection: mongodb+srv://user:password@cluster.mongodb.net/db
  
  - PostgreSQL: Recommended for traditional hosting
    Connection: postgresql://user:password@localhost:5432/db

TROUBLESHOOTING:
================================================================================

Server won't start:
  - Check if port 3000 is available
  - Check Node.js version (16+)
  - Check npm installation
  - Run: npm install

Tests failing:
  - Make sure server is running (npm run dev)
  - Check if port 3000 is accessible
  - Check firewall settings

API errors:
  - Check userId is correct
  - Check user has completed OAuth flow
  - Check environment variables
  - Check Google API credentials
